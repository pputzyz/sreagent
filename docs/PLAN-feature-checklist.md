# FlashCat 功能细节清单 — 逐条审计

> 逐条列出 FlashCat 文档中描述的所有功能点，标注是否纳入 SREAgent 实现计划。
> 标记说明：[Y] = 纳入实现 | [N] = 不做 | [E] = 已有（Existing）

---

## 一、故障模型 (Incident)

### 1.1 三层事件模型
- [Y] Event（原始事件）：监控系统每次触发/恢复产生一条 Event
- [Y] Alert（告警）：同一 alert_key 的多次 Event 合入同一条 Alert
- [Y] Incident（故障）：多条相似 Alert 聚合为一条 Incident

### 1.2 严重程度
- [Y] 三级：Info / Warning / Critical
- [Y] 故障严重程度 = 关联告警中最高级别
- [Y] 告警严重程度 = 关联事件中最高级别
- [Y] 严重程度可手动修改（修改后不随新告警变化）

### 1.3 处理进度 (Status)
- [Y] Triggered（待处理）→ Processing（处理中）→ Closed（已关闭）
- [Y] 任意人认领 → 进入 Processing
- [Y] 所有人取消认领 → 回退 Triggered
- [Y] 手动/自动关闭 → Closed

### 1.4 故障状态 (恢复状态)
- [Y] 未恢复：关联告警至少有一条未恢复
- [Y] 已恢复：关联告警均已恢复 → 故障自动关闭

### 1.5 故障标签
- [Y] 标签来自第一条告警
- [Y] 标签不可手动修改（仅来自事件上报）
- [Y] 标签用于筛选/检索/聚合/路由/分派/静默/抑制

### 1.6 故障生命周期
- [Y] 自动触发（告警事件触发）
- [Y] 手动创建（控制台创建故障，选择空间/严重程度/分派方式）
- [N] 外部提报（外部人员通过链接提交）— 暂不做
- [Y] 分派与通知（按分派策略）
- [Y] 认领（控制台/飞书卡片按钮）
- [Y] 暂缓（认领后暂缓升级，可配置时长预设 3 个）
- [Y] 关闭（手动/自动恢复/超时自动关闭）
- [Y] 重新打开（已关闭故障可重新打开，填写原因）
- [Y] 故障时间线（完整生命周期记录）

### 1.7 故障操作
- [Y] 认领 / 取消认领
- [Y] 暂缓（可配置 3 个预设时长，暂缓超时回退待处理）
- [Y] 关闭 / 重新打开
- [Y] 修改标题 / 严重程度 / 描述
- [Y] 合并故障（将当前故障并入目标故障）
- [Y] 重新分派（分派给其他人/值班表）
- [Y] 升级分派（按策略升级到下一环节）
- [Y] 添加处理人
- [Y] 快速静默（跳转到静默配置，预填当前故障信息）
- [Y] 批量操作（批量认领/批量关闭/批量重新打开）
- [N] 创建作战室 — 暂不做
- [N] 自定义操作（Jira/ServiceNow 同步）— 暂不做

### 1.8 故障详情页
- [Y] 顶部：标题 + 严重程度 + 处理进度 + 操作按钮
- [Y] Tab: 故障概览（描述/标签/AI 总结）
- [Y] Tab: 关联告警（聚合的告警列表，按处理进度筛选）
- [Y] Tab: 时间线（完整生命周期 + Markdown 评论框 + 图片粘贴）
- [Y] Tab: 故障复盘（内嵌编辑器 + AI 生成）
- [N] Tab: 历史变更（变更事件关联）— 暂不做
- [N] Tab: 历史故障（相似故障）— 暂不做
- [N] Tab: 问题修复（impact/root_cause/resolution 结构化字段）— Phase 5 考虑
- [Y] 右侧面板：属性 + 关键时间节点 + 处理人员 + 关联链接
- [N] 右侧面板：图片（告警附带图片）— 暂不做
- [N] 右侧面板：自定义字段 — 暂不做
- [N] 右侧面板：外部工单 — 暂不做

### 1.9 AI 能力
- [Y] AI 故障总结（一键生成：概述 + 影响 + 建议）
- [Y] AI 辅助复盘（自动生成复盘报告初稿）

## 二、协作空间 (Channel)

### 2.1 基本信息
- [Y] 空间名称 + 描述
- [Y] 所属团队（team_id）
- [N] 代管团队 — 暂不做
- [Y] 访问级别：公开 / 私有
- [Y] 超时自动关闭（开关 + 计时起点 + 超时时长）
- [Y] 跟随告警关闭（关联告警全部恢复 → 自动关闭故障）
- [N] 新奇故障检测 — 暂不做
- [N] 外部提报 — 暂不做

### 2.2 空间列表
- [Y] 卡片式列表（空间名/团队/活跃故障数/MTTA/MTTR）
- [Y] 收藏（星标）
- [Y] 按团队筛选
- [Y] 排序（创建时间/最新故障时间/名称/自定义拖拽顺序）
- [Y] 创建空间向导（基本信息 → 分派策略 → 集成接入，后两步可跳过）

### 2.3 空间详情 — 故障列表 Tab
- [Y] 处理进度筛选
- [Y] 批量关闭 / 批量认领
- [Y] 合并功能（跨空间合并）

### 2.4 空间详情 — 统计概览
- [Y] MTTA / MTTR / 故障数 / 告警分组 四张统计卡片
- [Y] 基于最近 7 天，与上一周期同比

### 2.5 空间详情 — 配置 Tab
- [Y] 集成数据：专属 Webhook 端点
- [Y] 集成数据：排除规则（按集成来源/严重程度过滤，命中即丢弃）
- [Y] 降噪处理：告警聚合（规则聚合，按标签维度）
- [Y] 降噪处理：聚合窗口（开关 + 计时起点 + 时长）
- [Y] 降噪处理：风暴预警（合入告警数达阈值通知）
- [Y] 降噪处理：抖动检测（状态变化次数 + 观测窗口 + 静默时长）
- [Y] 降噪处理：静默策略（单次/周期-星期模式，支持快速静默）
- [N] 降噪处理：静默策略 — 日历模式（依赖服务日历）— 暂不做
- [Y] 降噪处理：抑制策略
- [Y] 通知分派：分派策略（绑定到空间）
- [Y] 设置：基础信息编辑 + 高级配置
- [N] 关联链接 (Links)：模板变量动态 URL — 暂不做（Phase 5 考虑）
- [N] 智能聚合（机器学习语义相似度）— 暂不做，先做规则聚合

## 三、告警管理 (Alert)

### 3.1 告警列表页
- [Y] 列：严重程度/标题/处理状态/故障处理进度/关联事件数/关联故障/触发时间/集成来源/协作空间
- [Y] 筛选：标题/严重程度/协作空间/集成/标签/是否被静默或抑制
- [Y] 视图设置：详情展示模式(页面/右侧面板)、自动刷新、展示属性
- [Y] 批量操作：并入故障
- [Y] 聚合视图（按维度 Group By）

### 3.2 告警详情页
- [Y] HeaderInfo：标题 + 严重程度 + ID + 恢复状态 + 故障处理进度
- [Y] Tab: 告警概览（描述/标签/关联事件预览/属性/来源/所属故障）
- [Y] Tab: 关联事件（原始事件列表）
- [Y] 并入操作（将告警并入指定故障）

### 3.3 告警生命周期记录
- [Y] a_new: 新触发
- [Y] a_update: 更新
- [Y] a_merge: 合并至故障
- [Y] a_m_silence: 命中静默
- [Y] a_m_inhibit: 命中抑制

## 五、分派策略 (Escalation)

### 5.1 策略配置
- [Y] 绑定到协作空间
- [Y] 多条策略按优先级从上到下匹配，匹配即停
- [Y] 启用/禁用策略
- [Y] 策略复制（到当前空间或其他空间）
- [Y] 拖动调整策略顺序

### 5.2 触发条件
- [Y] 生效时间：7x24 / 特定时间段（如仅工作日 9:00-18:00）
- [N] 生效时间：服务日历模式 — 暂不做
- [Y] 故障筛选：按标题/级别/标签条件匹配

### 5.3 通知对象
- [Y] 值班表（当前正在值班的人）
- [Y] 团队（整个团队所有成员）
- [Y] 个人（固定人员）
- [Y] 组合模式

### 5.4 通知方式
- [Y] 单聊：邮件 + 飞书私聊
- [Y] 群聊：飞书群机器人
- [Y] 遵循个人偏好 vs 遵循统一设置
- [N] 电话/短信 — 暂不做
- [N] 钉钉/企微/Slack/Teams — 暂不做

### 5.5 延迟窗口
- [Y] 0-3600 秒，默认 0
- [Y] 延迟期内故障关闭则不发送通知

### 5.6 通知模板
- [E] 已有 MessageTemplate 模块
- [Y] 分派策略必须选择通知模板

### 5.7 升级规则
- [Y] 多个分派环节（默认首环节 30 分钟超时）
- [Y] 增删环节 / 上下移动 / 插入
- [Y] 升级条件：未关闭 N 分钟 / 未关闭且未认领 N 分钟
- [Y] 重复通知（可配置间隔和次数）
- [Y] 自动升级到下一环节
- [Y] 手动升级到指定环节

## 六、告警降噪

### 6.1 告警聚合
- [Y] 规则聚合：按标签维度精确匹配（最多 5 个维度/组）
- [Y] 统一控制：空间内所有告警用同一维度
- [Y] 细粒度控制：按条件分支匹配不同聚合维度（最多 100 条分支）
- [N] 智能聚合（机器学习语义相似度）— 暂不做
- [Y] 聚合窗口：开关 + 时长（最小 1 分钟，最大 24 小时）
- [Y] 窗口计时起点：故障触发 / 告警合入故障
- [Y] 风暴预警：阈值配置（最多 5 个，范围 2-10000）
- [Y] 严格聚合：空值标签视为不同项 vs 相同项

### 6.2 抖动检测
- [Y] 关闭 / 仅提醒 / 提醒后静默
- [Y] 参数：状态变化次数(2-100) + 观测窗口(1-1440分钟) + 静默时长(30-1440分钟)

### 6.3 静默策略
- [Y] 单次静默（指定时间段）
- [Y] 周期静默 — 星期模式（如每周六 00:00-06:00）
- [N] 周期静默 — 日历模式 — 暂不做
- [Y] 静默条件：严重程度 / 标题 / 描述 / 集成来源 / 标签（AND/OR 组合）
- [Y] 静默行为：直接丢弃 / 保留标记
- [Y] 快速静默（从故障详情一键创建，预填条件，默认 1 天）

### 6.4 抑制策略
- [E] 已有 InhibitionRule
- [Y] 新告警条件 + 活跃告警条件 + 相同项
- [Y] 抑制行为：直接丢弃 / 保留标记
- [Y] 匹配窗口：最近 10 分钟内活跃的源告警

## 七、告警处理管道 (Pipeline)

### 7.1 执行顺序
- [Y] 标签增强 → 告警处理 → 路由分发

### 7.2 核心功能
- [Y] 严重程度自定义（条件匹配 → 重写严重程度）
- [Y] 信息重写（标题/描述，支持 Go Template 语法引用标签）
- [Y] 告警丢弃（条件匹配直接丢弃）
- [Y] 告警抑制（Pipeline 层级抑制，仅匹配同一集成内告警）
- [Y] 链式处理（多条规则从上到下依次执行）
- [Y] 恢复事件不经过 Pipeline（直接合入）

### 7.3 模板语法
- [Y] `[TPL]{{.Labels.xxx}}` — 引用标签
- [Y] `[TPL]{{.Title}}` — 引用标题
- [Y] `[TPL]{{.Description}}` — 引用描述
- [Y] `[TPL]{{.EventSeverity}}` — 引用当前等级
