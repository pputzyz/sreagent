# SREAgent 核心流程测试指南

> v4.44.5 | 2026-05-27
>
> 本手册提供可操作的测试用例，以 admin 身份登录后逐步验证平台核心功能。

## 前置条件

1. 服务已启动: `make dev` 或 `make run`
2. 浏览器访问 `http://localhost:8080`
3. 使用 admin 账号登录
4. 打开浏览器 DevTools (F12) → Console 标签，观察错误

## 常见问题排查

### 问题：页面按钮全部消失

**原因**: `usePermissions.ts` 的 `hasPerm()` 在 API 返回前返回 false。

**检查**:
1. 打开 Console，看是否有 `Failed to load permissions` 警告
2. 检查 Network 标签，`/api/v1/me/permissions` 是否返回 200
3. 确认 `localStorage.getItem('user_role')` 返回正确角色

### 问题：OIDC 白屏

**原因**: vue-i18n 的 `SyntaxError`，通常是 i18n 文件中有未转义的 `{}`。

**检查**: Console 中搜索 `SyntaxError`，定位到 i18n 文件的哪一行。

### 问题：告警事件不触发

**检查**:
1. 数据源是否健康（绿色状态）
2. 告警规则状态是否为 active
3. PromQL 表达式是否正确（在数据查询中验证）
4. 后端日志是否有引擎评估错误
5. `for_duration` 是否设置过长

### 问题：通知不发送

**检查**:
1. 通知渠道配置是否正确（测试发送）
2. 通知策略是否匹配告警的严重等级和标签
3. 是否被静默规则屏蔽
4. 后端日志搜索 `notify` 查看发送详情
